# PR-15 — Greentic-dev E2E: local store & offline mode

## Goal
Prove that greentic-dev works **fully offline** using a local component/pack store.

## Scope
- No network access during tests.
- Explicitly configure local store paths.
- Validate install → build → run entirely from local artifacts.

## Test cases to add

### 1) Local component store install
- Configure a local store directory via env or CLI flag.
- Scaffold and build a component.
- Install it into the local store.
- Assert:
  - component appears in `greentic-dev component list`
  - no network calls are attempted.

### 2) Pack build using only local store
- Generate a new pack.
- Reference components that exist only in the local store.
- Run:
  - `greentic-dev pack validate`
  - `greentic-dev pack build`
- Assert build succeeds without network access.

### 3) Offline run
- Explicitly enable offline mode (flag or env).
- Run the pack via `greentic-dev`.
- Assert:
  - execution succeeds
  - output matches expected result
  - logs confirm offline mode.

## Structure
- Add new test module:
  - `tests/e2e_greentic_dev_offline.rs`
- Reuse workspace/bootstrap helpers from PR-13.

## Acceptance criteria
- All tests pass with networking disabled.
- Local store behavior is deterministic and documented.
